# Bd UI 规范

## 设计原则

- 采用 Material You 的组件语义、动态色能力、状态层与无障碍触控尺寸。
- 内容优先。仅在“添加下载”和独立任务上使用容器，设置项采用分组列表，避免每一行都套卡片。
- 以 8dp 为基础网格；页面水平边距 20dp，列表容器边距 16dp，主要触控区不低于 48dp。
- 主标题 22–32sp，分组标题 16sp，正文 14–16sp；中文使用系统无衬线字体族，避免额外打包大型 CJK 字体。
- 珊瑚红仅用于主要操作、选中态和进度；错误态使用系统 error 色，不用颜色作为唯一状态提示。
- 缩略图统一 3:2 左右比例；文本、进度和操作以共同基线对齐。

## 信息架构

1. 首页：链接输入、识别、最近任务、支持范围。
2. 任务：进行中与已完成分栏，展示速度、字节、进度、错误与恢复操作。
3. 设置：账号、媒体偏好、存储、默认资源、弹幕与关于。
4. 任务配置底部面板：内容摘要、选集、资源开关、最终确认。
5. 登录：独立全屏 WebView，始终提供返回与完成操作。

## 视觉验收

- 长标题必须截断，不挤压操作按钮。
- 空状态、加载、错误、暂停、完成均有文本反馈。
- 深浅色模式使用系统语义色，关键文字对比度由 Material 3 色板保证。
- 所有开关的行区域均可点击；图标按钮提供 contentDescription。
- 底部面板在小屏设备上可滚动，主要按钮不会被手势导航区遮挡。
